# Circul-AI-r Platform — Render Deployment Guide

This document provides complete instructions for deploying the Circul-AI-r Platform on [Render](https://render.com) with auto-deployment from GitHub.

---

## Architecture Overview

```
GitHub (main branch)
    │
    ▼ auto-deploy on push
Render Web Service (Node.js)
    │
    ├── Express + tRPC API server
    ├── Vite-built React SPA (served as static files)
    ├── Socket.io (real-time telemetry)
    └── MQTT subscriber (IoT ingestion)
    │
    ▼ external services
    ├── MySQL/TiDB (database)
    ├── AWS S3 (file storage)
    ├── OpenAI (AI/LLM)
    ├── ZeptoMail (email)
    ├── Stripe (payments)
    ├── EMQX (MQTT broker)
    └── Google Maps (geolocation)
```

---

## Prerequisites

1. **GitHub repository:** `HarishBoke/circulair-platform` (already configured)
2. **Render account:** [render.com](https://render.com)
3. **External MySQL database** (TiDB Cloud recommended, or any MySQL 8+ compatible)
4. **AWS S3 bucket** for file storage
5. **OpenAI API key** for AI features
6. **ZeptoMail account** for transactional email
7. **Stripe account** for payments (optional)
8. **EMQX Cloud** or any MQTT broker for IoT telemetry (optional)

## Quick Start (Render Blueprint)

The fastest way to deploy:

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **New → Blueprint**
3. Connect the `HarishBoke/circulair-platform` GitHub repository
4. Render will auto-detect `render.yaml` and create the service
5. Fill in the required environment variables (see below)
6. Click **Apply**

---

## Manual Setup (Alternative)

If you prefer manual configuration:

1. **New → Web Service**
2. Connect GitHub repo: `HarishBoke/circulair-platform`
3. Configure:
   - **Name:** `circulair-platform`
   - **Region:** Singapore (or closest to your users)
   - **Branch:** `main`
   - **Runtime:** Node
   - **Build Command:** `pnpm install --frozen-lockfile && pnpm build`
   - **Start Command:** `node dist/index.js`
   - **Plan:** Starter ($7/mo) or Standard ($25/mo)
4. Add environment variables (see below)
5. Deploy

## Database Setup

The platform uses **MySQL** (TiDB-compatible). You need an external MySQL database.

### Recommended: TiDB Cloud (Serverless)

1. Create a free cluster at [tidbcloud.com](https://tidbcloud.com)
2. Get the connection string from **Connect → General**
3. Format: `mysql://user:pass@gateway.tidbcloud.com:4000/circulair?ssl={"rejectUnauthorized":true}`

### Alternative: PlanetScale, AWS RDS, or any MySQL 8+

Any MySQL-compatible database works. Ensure SSL is enabled for production.

### Running Migrations

After setting `DATABASE_URL`, run migrations:

```bash
# From your local machine with the same DATABASE_URL
pnpm db:push
```

Or use the Render Shell:
```bash
cd /opt/render/project/src && pnpm db:push
```

## Auto-Deployment

Once connected to GitHub:

1. Every push to `main` triggers a new deployment
2. Render runs `pnpm install --frozen-lockfile && pnpm build`
3. On success, the new version goes live with zero-downtime rollout
4. Failed builds do not affect the running service

### Health Check

The service exposes `/api/health` which Render uses to verify the deployment is healthy before routing traffic.

## Custom Domain

1. In Render Dashboard → your service → **Settings → Custom Domains**
2. Add `circulair.energy` (or your domain)
3. Add the CNAME/A record shown by Render to your DNS provider
4. SSL is automatically provisioned via Let's Encrypt

---

## Stripe Webhook Setup

After deploying, configure the Stripe webhook:

1. Go to [Stripe Dashboard → Developers → Webhooks](https://dashboard.stripe.com/webhooks)
2. Add endpoint: `https://your-domain.com/api/stripe/webhook`
3. Select events: `checkout.session.completed`, `payment_intent.succeeded`, `invoice.paid`
4. Copy the signing secret → set as `STRIPE_WEBHOOK_SECRET` in Render

## Admin Bootstrap

To promote the first admin user after deployment:

```bash
curl -X POST https://your-domain.com/api/admin/bootstrap-admin \
  -H "Content-Type: application/json" \
  -d '{"secret": "YOUR_ADMIN_BOOTSTRAP_SECRET", "email": "admin@circulair.energy"}'
```

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Build fails with `pnpm` error | Ensure Node.js 22+ is set in Render settings |
| Database connection timeout | Check `DATABASE_URL` includes `?ssl={"rejectUnauthorized":true}` |
| Login not working | Verify `JWT_SECRET` is set and consistent across deploys |
| MQTT not connecting | Check `MQTT_BROKER_URL` uses `mqtts://` (TLS) and credentials are correct |
| Static files not served | Ensure `pnpm build` completed successfully (check build logs) |
| Socket.io not working | Render supports WebSockets on all plans — no extra config needed |

## File Structure (Production Build)

```
dist/
  index.js        ← Bundled Express server (esbuild)
  public/         ← Vite-built React SPA
    index.html
    assets/       ← JS/CSS bundles
```

The `pnpm build` command:
1. Runs `vite build` → outputs client to `dist/public/`
2. Runs `esbuild` → bundles server to `dist/index.js`

The production server (`node dist/index.js`) serves both the API and the static SPA.
